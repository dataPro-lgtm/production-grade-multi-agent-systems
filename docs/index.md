# 生产级多智能体系统

## 从架构判断到工程落地

多 Agent 系统最难的部分，不是创建几个角色或让模型互相发送消息，而是做出一系列可以被证明的工程判断：

- 这个任务是否真的需要 Agent？
- 自治应该停在哪一级？
- 多个责任单元之间是否存在真实的领域、数据、工具、权限、状态或故障边界？
- 当模型、工具、网络和外部数据都可能出错时，系统如何安全地继续、恢复或停止？

本书沿着“判断 → 设计 → 实现 → 验证 → 运营”的路径，逐步建立一套生产级方法。

## 你将获得什么

| 层次 | 核心问题 | 可复用产物 |
|---|---|---|
| 架构判断 | 工作流、单 Agent 还是多 Agent？ | 适用性评分、ADR |
| 边界设计 | 怎样拆分才有工程价值？ | 领域、工具、权限与状态契约 |
| 运行机制 | Agent 如何观察、决策、行动和更新？ | 状态机、工具协议、终止条件 |
| 生产控制 | 怎样限制风险并处理失败？ | 审批、幂等、重试、回滚 |
| 质量验证 | 如何证明系统可以上线？ | Golden Dataset、回归门禁、Trace |
| 持续运营 | 如何控制成本、延迟和演进复杂度？ | SLO、成本预算、版本策略 |
