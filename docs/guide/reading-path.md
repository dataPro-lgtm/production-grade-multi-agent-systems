# 阅读路径

这本书既可以顺序阅读，也可以按职责进入。

## 顺序阅读

适合第一次系统学习多 Agent 工程的读者：

1. 先建立 Agent 与工作流的判定标准；
2. 再理解工具调用、状态与反馈闭环；
3. 学习多 Agent 的拆分边界与协作模式；
4. 进入安全、评测、可观测性和故障恢复；
5. 最后通过完整案例把判断落到代码与上线门禁。

## 按角色阅读

| 角色 | 建议重点 |

| AI / LLM 工程师 | 第 02、04、06、07、09、10、12 章 |
| 数据工程师 | 第 04、08、09、12、13 章 |
| 产品技术负责人 / FDE | 第 01、03、05、10、11、14 章 |

## 每章的固定学习闭环

每章尽量遵循同一结构：

1. 真实业务问题；
2. 核心概念与常见误判；
3. 候选架构与取舍；
4. 可执行契约或模板；
5. 文件级代码路径与验证证据（涉及实现时）；
6. 失败场景与安全停止；
7. 练习、参考答案和上线检查。
