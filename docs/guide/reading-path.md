# 阅读路径

这本书既可以顺序阅读，也可以按职责进入。

## 顺序阅读

适合第一次系统学习多 Agent 工程的读者：

1. 先建立 Agent 与工作流的判定标准；
2. 再理解工具调用、状态与反馈闭环；
3. 学习多 Agent 的拆分边界与协作模式；
4. 建立 Context Engineering、混合检索、GraphRAG 与证据治理能力；
5. 把系统部署为具备健康门禁、可观测性、弹性和恢复闭环的平台；
6. 建立不可被自然语言绕过的授权、隐私、审计和红队安全边界；
7. 把 Supervisor、A2A、MCP、状态与 Context Graph 合成一条可验收链路；
8. 用 Golden Dataset、分层指标、Judge、N-run 与持续回归证明系统质量；
9. 用 AgentOps 把目标级 SLO、因果诊断、运行控制、韧性、成本与持续优化连成闭环；
10. 最后用 Capstone、Evidence Package、Release Gates 和独立复现完成系统验收与开源交付。

## 每章的固定叙事结构

每章尽量遵循同一结构：

1. 真实业务问题；
2. 核心概念与常见误判；
3. 候选架构与取舍；
4. 可执行契约或模板；
5. 文件级代码路径与验证证据（涉及实现时）；
6. 失败场景与安全停止；
7. 可以直接用于工程决策的结论与检查方法。
