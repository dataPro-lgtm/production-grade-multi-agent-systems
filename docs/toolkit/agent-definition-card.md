# Agent 定义卡

在创建 Agent 之前，先填写这张卡。无法明确回答的字段，通常意味着边界还没有设计完成。

| 字段 | 填写内容 |
|---|---|
| Agent 名称 | 用责任域命名，避免只写人格名称 |
| 业务目标 | 它为谁解决什么问题 |
| 成功标准 | 如何判断已完成且正确 |
| 允许输入 | 接受哪些任务、上下文和身份 |
| 允许输出 | 必须返回什么结构和证据 |
| 允许工具 | 可调用工具及其读写属性 |
| 数据边界 | 可以访问哪些数据 |
| 权限边界 | 哪些动作禁止或需要审批 |
| 状态 | 需要持久化哪些任务状态 |
| 预算 | 时间、Token、费用、重试上限 |
| 终止条件 | 成功、失败、转人工和拒答条件 |
| 评测 | 用哪些数据集和指标验证 |
| Owner | 谁维护、审批和响应故障 |

## 最小示例

```yaml
name: refund-analysis-agent
goal: 识别指定范围内退款率变化的主要驱动因素
success:
  - 所有结论都有查询证据
  - 分母、时间和过滤条件明确
allowed_tools:
  - metadata_search
  - read_only_sql
forbidden:
  - 写入生产数据库
  - 绕过行列级权限
stop:
  - 证据达到发布门槛
  - 预算耗尽
  - 权限不足
  - 需要人工澄清
```
