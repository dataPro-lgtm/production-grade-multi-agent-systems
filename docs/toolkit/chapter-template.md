# 第 NN 章：标题

> 状态：提纲 / 初稿 / 技术审校 / 可发布

## 本章要解决的问题

用一个真实、具体、可验证的业务或工程问题开场。

## 学习目标

完成本章后，读者能够：

1. …
2. …
3. …

## 核心判断

给出概念、边界、常见误判和最小判断标准。

## 候选架构与取舍

至少比较两个方案，说明收益、代价、适用条件和回退方式。

## 代码路径

涉及代码时，按照真实执行顺序逐文件展开：

1. 文件用途；
2. 关键导入和依赖；
3. 核心函数或类；
4. 调用链与状态变化；
5. 测试如何证明行为；
6. 失败、重试、权限和停止路径。

## 实战

提供可以运行的输入、命令、预期输出和验收标准。

## 失败模式

至少覆盖正常错误、高风险错误和证据不足。

## 练习与参考答案

练习应要求读者做判断，而不仅是复述定义。

## 本章检查清单

- [ ] 架构选择有明确理由
- [ ] 模型决策与确定性控制分开
- [ ] 状态、权限和终止条件明确
- [ ] 代码与测试可以对应
- [ ] 第三方内容已引用

## 参考资料

优先引用规范、官方文档和原始论文。
