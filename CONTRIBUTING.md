# 贡献指南

感谢你帮助完善《生产级多智能体系统：从架构判断到工程落地》。

## 可以贡献什么

- 错字、病句、失效链接和事实勘误；
- 更清晰的架构图、案例和失败路径；
- 可运行的实验、测试和评测数据；
- 新章节提案或现有章节重写；
- 无障碍、术语一致性和阅读体验改进。

## 开始之前

1. 小型勘误可以直接提交 Pull Request；
2. 新章节、较大重构或许可相关修改，请先创建 Issue；
3. 不要提交受版权保护且未获授权的书稿、课程、图片或代码；
4. 不要在示例中放入密钥、真实个人信息或生产数据。

## 章节质量标准

每章应尽可能包含：

- 一个真实问题，而不是只介绍概念；
- 明确的架构判断和候选方案比较；
- 数据、工具、权限、状态与故障边界；
- 可复制的契约、清单或模板；
- 练习、参考答案与失败场景；
- 可核验的参考资料。

涉及代码讲解时，请沿真实执行路径按文件展开。对每个关键文件说明用途、导入、核心函数或类、调用流、测试证据和失败路径。不要只画一张宏观架构图就结束。

## 本地检查

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
./scripts/validate.sh
```

## 文件命名

- 章节：`NN-short-english-slug.md`
- 图片：小写英文与连字符，例如 `agent-minimal-loop.png`
- ADR：`ADR-NNN-short-title.md`
- 图片放在对应章节目录，例如 `docs/assets/images/chapter-01/`

## Pull Request 清单

- [ ] 我已运行 `./scripts/validate.sh`
- [ ] 新增图片有清晰替代文本
- [ ] 新增代码有复现命令和测试
- [ ] 引用与第三方许可已核对
- [ ] 术语与全书保持一致
